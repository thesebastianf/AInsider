# AInsider Tracker – Database Migrations Guide

This project uses **Alembic** (built on top of SQLAlchemy) to manage database schema updates.

---

## 1. How Migrations Work in this Project

*   **Initial Setup:** All tables are created by the initial migration: [001_initial.py](file:///c:/DEV/AInsider/backend/alembic/versions/001_initial.py).
*   **Automatic Upgrades:** When running via Docker, the backend container automatically executes `alembic upgrade head` on startup before launching the FastAPI application.

---

## 2. Creating New Migrations

If you modify models in `backend/app/models.py`, you must generate a new migration script:

1.  **Generate Autogenerated Revision:**
    Run the following command from the host or within the running backend container to detect model differences and create a script:
    ```bash
    # From inside the running backend docker container:
    docker-compose exec app alembic revision --autogenerate -m "describe_your_changes"
    ```
2.  **Review the Generated File:**
    A new Python script will be created under `backend/alembic/versions/`. Always verify that `upgrade()` and `downgrade()` match your changes.

---

## 3. Standard Migration Commands

Run these inside the container (or locally if your python environment points to the database):

*   **Upgrade to the latest schema version:**
    ```bash
    alembic upgrade head
    ```
*   **Rollback the last migration:**
    ```bash
    alembic downgrade -1
    ```
*   **Check migration history:**
    ```bash
    alembic history
    ```

---

## 4. Resetting/Re-seeding the Database (Development)

If database tables get out of sync or migrations get corrupted during local development:

```bash
# 1. Stop all containers and wipe the database volume
docker-compose down -v

# 2. Start the database and build/run the application
# This automatically runs alembic upgrade head and seeds default data
docker-compose up --build -d
```
